Краткая инструкция по использованию парсера:
1)Ничего не трогать! Exe файл сам запускается планировщиком задач. 
2)Особенно нельзя трогать old-ids.txt! Он испольщуется для хранения спарсенных id и для проверки на новизну. Если удалить хотя бы строчку, при след. парсинге оффер отправится в crm повторно.
3)Исходный код находится в parser.py
4)Не менять ссылки на crm форму
5)Не менять crm форму
---------------------------------
Описание функций.
---------------------------------
get_main_dict() - возвращает паттерн словаря оффера.
get_new_ids() - возвращает список новых id. 
get_old_ids(url) - возвращает список старых id из old-ids.txt. url = 'old-ids.txt'
ids_validation(old_ids_url) - сранивает результаты функций get_new_ids() и get_old_ids(url) и возвращает уникальные id, которые раньше не встречались
get_offer_by_id(id) - возвращает заполненный словарь, заполненный данными с оффера.
download_file(url) - загружает xml документ для парсинга
send_offer(offer) - отправляет словарь в crm форму. offer - заполнный словарь, который выдается функцией get_offer_by_id(id)
